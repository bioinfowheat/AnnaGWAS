MultiQC Quality Control Strategy

MultiQC (multiqc.info) aggregates quality control metrics from multiple bioinformatics tools into a single interactive HTML report. This workflow includes 4 QC checkpoints that generate MultiQC reports at critical stages:

Checkpoint Stage Tools Aggregated Key Metrics
QC1 Read Quality FastQC, fastp Read quality, adapter content, duplication
QC2 Alignment samtools stats/flagstat/idxstats, Picard Mapping rate, coverage, duplicates
QC3 Variants bcftools stats, VCFtools Variant counts, Ti/Tv ratio, quality distribution
QC4 GWAS Ready PLINK, custom scripts Sample QC, HWE, missingness, PCA

MultiQC Configuration File

Create a multiqc_config.yaml file in your project directory to customize reports:

# multiqc_config.yaml
title: "GWAS Pipeline QC Report"
subtitle: "96 samples - Annas_GWAS project"

# Report customization
report_header_info:
    - Project: "Annas_GWAS"
    - Samples: "96"
    - Phenotype: "Binary (0/1)"

# Module order (customize per checkpoint)
module_order:
    - fastqc
    - fastp
    - samtools
    - picard
    - bcftools

# Sample name cleaning
extra_fn_clean_exts:
    - ".dedup"
    - ".sorted"
    - ".trimmed"
    - "_R1"
    - "_R2"

# Table columns to show
table_columns_visible:
    FastQC:
        percent_duplicates: True
        percent_gc: True
        avg_sequence_length: True
        total_sequences: True
    Samtools:
        mapped_passed: True
        error_rate: True

# Export data for downstream use
export_plots: true
data_format: "tsv"

3. Phase 1: Read Quality Control

Objective

Assess and improve raw read quality, remove adapters and low-quality bases.

Step 1.1: Initial Quality Assessment

# Create output directories
mkdir -p qc/fastqc_raw qc/fastqc_clean qc/fastp

# Run FastQC on raw reads (all samples in parallel)
fastqc -t 24 -o qc/fastqc_raw/ data/fastq/*.fastq.gz

Step 1.2: Read Trimming with fastp

# Process each sample
for sample in $(ls data/fastq/*_R1.fastq.gz | xargs -n1 basename | sed 's/_R1.fastq.gz//'); do
    fastp \
        -i data/fastq/${sample}_R1.fastq.gz \
        -I data/fastq/${sample}_R2.fastq.gz \
        -o results/trimmed/${sample}_R1.trimmed.fastq.gz \
        -O results/trimmed/${sample}_R2.trimmed.fastq.gz \
        --detect_adapter_for_pe \
        --qualified_quality_phred 20 \
        --length_required 50 \
        --correction \
        --thread 8 \
        --json qc/fastp/${sample}.fastp.json \
        --html qc/fastp/${sample}.fastp.html
done

Step 1.3: Post-trimming Quality Check

# Run FastQC on trimmed reads
fastqc -t 24 -o qc/fastqc_clean/ results/trimmed/*.fastq.gz
QC CHECKPOINT 1 Read Quality Report

Generate MultiQC Report

# Generate QC1 report
multiqc qc/fastqc_raw qc/fastqc_clean qc/fastp \
    -o reports/qc1_reads \
    -n multiqc_reads.html \
    -c multiqc_config.yaml \
    --title "QC1: Read Quality Assessment"

What to Check in the Report

Metric Expected Action if Failed
Per base sequence quality Q20+ across all positions Trim more aggressively
Per sequence quality scores Peak at Q30+ Check sequencing run quality
Adapter content < 5% after trimming Re-run fastp with explicit adapters
Sequence duplication < 30% (WGS) Note for later; duplicates marked post-alignment
GC content Matches expected genome GC Check for contamination
Reads passing filters (fastp) > 90% Investigate low-quality samples

Decision Point

STOP if:

  • More than 10% of samples have < 80% reads passing filters
  • Systematic quality issues across all samples (sequencing problem)
  • Unexpected GC distribution suggesting contamination

4. Phase 2-3: Alignment and Post-Processing

Objective

Align reads to reference genome and prepare analysis-ready BAM files.

Step 2.1: Index Reference

# Index reference genome (one time)
bwa index reference.fasta
samtools faidx reference.fasta

Step 2.2: Align Reads

# Align each sample
for sample in $(cat sample_list.txt); do
    bwa mem -t 16 -M \
        -R "@RG\tID:${sample}\tSM:${sample}\tPL:ILLUMINA\tLB:lib1" \
        reference.fasta \
        results/trimmed/${sample}_R1.trimmed.fastq.gz \
        results/trimmed/${sample}_R2.trimmed.fastq.gz \
    | samtools view -bS - \
    | samtools sort -@ 8 -o results/aligned/${sample}.sorted.bam -
done

Step 2.3: Mark Duplicates

# Mark duplicates with samtools (generates metrics)
for sample in $(cat sample_list.txt); do
    samtools markdup -@ 8 \
        -f qc/dedup/${sample}.markdup_metrics.txt \
        results/aligned/${sample}.sorted.bam \
        results/aligned/${sample}.dedup.bam

    samtools index results/aligned/${sample}.dedup.bam
done

Step 2.4: Generate Alignment Statistics

# Generate comprehensive statistics for each sample
mkdir -p qc/alignment

for sample in $(cat sample_list.txt); do
    # Detailed stats
    samtools stats results/aligned/${sample}.dedup.bam \
        > qc/alignment/${sample}.stats

    # Quick flagstat summary
    samtools flagstat results/aligned/${sample}.dedup.bam \
        > qc/alignment/${sample}.flagstat

    # Per-chromosome stats
    samtools idxstats results/aligned/${sample}.dedup.bam \
        > qc/alignment/${sample}.idxstats

    # Coverage statistics (optional but useful)
    samtools coverage results/aligned/${sample}.dedup.bam \
        > qc/alignment/${sample}.coverage
done
QC CHECKPOINT 2 Alignment Quality Report

Generate MultiQC Report

# Generate QC2 report
multiqc qc/alignment qc/dedup \
    -o reports/qc2_alignment \
    -n multiqc_alignment.html \
    -c multiqc_config.yaml \
    --title "QC2: Alignment Quality Assessment"

What to Check in the Report

Metric Expected Action if Failed
Mapped reads > 95% Check reference genome, contamination
Properly paired > 90% Check library prep, insert size
Duplication rate < 30% (15X WGS) May indicate over-amplification
Insert size Expected library size (300-500bp typical) Library QC issue if bimodal
Error rate < 1% Sequencing quality issue
Coverage uniformity Low coefficient of variation GC bias, capture issues (not expected for WGS)

Coverage Analysis

# Calculate mean coverage per sample
echo -e "Sample\tMean_Coverage\tMedian_Coverage" > qc/coverage_summary.txt
for sample in $(cat sample_list.txt); do
    cov=$(samtools depth results/aligned/${sample}.dedup.bam | \
          awk '{sum+=$3; n++} END {print sum/n}')
    echo -e "${sample}\t${cov}" >> qc/coverage_summary.txt
done

# Verify target coverage (15X)
awk '$2 < 10 {print "WARNING: Low coverage sample:", $1, $2"X"}' qc/coverage_summary.txt

Decision Point

STOP if:

  • Any sample has < 80% mapping rate
  • Mean coverage < 10X for more than 10% of samples
  • Duplication rate > 50% across multiple samples

Consider removing samples with:

  • Coverage < 8X (may increase missing genotypes)
  • Duplication > 40% (effective coverage too low)

5. Phase 4-5: Variant Calling and Filtering

Objective

Call variants across all samples and apply quality filters.

Step 4.1: Create BAM List

# Create list of BAM files for joint calling
ls results/aligned/*.dedup.bam > bam_list.txt

Step 4.2: Joint Variant Calling (FreeBayes)

# Joint calling
freebayes -f reference.fasta \
    -L bam_list.txt \
    --min-mapping-quality 20 \
    --min-base-quality 20 \
    --min-alternate-count 2 \
    --min-alternate-fraction 0.1 \
    > results/variants/raw.vcf

# Compress and index
bgzip -c results/variants/raw.vcf > results/variants/raw.vcf.gz
tabix -p vcf results/variants/raw.vcf.gz

# Generate stats on RAW variants
bcftools stats results/variants/raw.vcf.gz > qc/variants/raw.bcftools_stats.txt

Step 4.3: Variant Filtering

# Apply quality filters
bcftools filter -i 'QUAL>20 && INFO/DP>50 && INFO/DP<5000' \
    results/variants/raw.vcf.gz | \
bcftools view -m2 -M2 -v snps | \
bgzip -c > results/variants/filtered.vcf.gz

tabix -p vcf results/variants/filtered.vcf.gz

# Generate stats on FILTERED variants
bcftools stats results/variants/filtered.vcf.gz > qc/variants/filtered.bcftools_stats.txt

# Per-sample stats
bcftools stats -s - results/variants/filtered.vcf.gz > qc/variants/filtered_persample.bcftools_stats.txt

Step 4.4: Generate Additional Variant Metrics

# Site frequency spectrum
bcftools query -f '%CHROM\t%POS\t%AF\n' results/variants/filtered.vcf.gz > qc/variants/allele_frequencies.txt

# Transition/Transversion ratio per sample
bcftools stats -s - results/variants/filtered.vcf.gz | \
    grep "TSTV" > qc/variants/tstv_ratio.txt

# Missing data per sample
bcftools stats -s - results/variants/filtered.vcf.gz | \
    grep "PSC" > qc/variants/per_sample_counts.txt
QC CHECKPOINT 3 Variant Quality Report

Generate MultiQC Report

# Generate QC3 report
multiqc qc/variants \
    -o reports/qc3_variants \
    -n multiqc_variants.html \
    -c multiqc_config.yaml \
    --title "QC3: Variant Quality Assessment"

What to Check in the Report

Metric Expected Action if Failed
Total variants (raw) ~1-5 million for 350Mb genome at 15X Check calling parameters
SNPs retained after filtering > 70% of raw SNPs Filters may be too strict
Ti/Tv ratio ~2.0-2.1 (coding), ~2.5 (genome-wide) Low Ti/Tv indicates false positives
Per-sample heterozygosity Consistent across samples Outliers may indicate contamination
Missing rate per sample < 5% Low coverage samples
Quality score distribution Peak at high quality values Adjust quality threshold

Decision Point

Warning signs:

  • Ti/Tv ratio < 1.8 suggests many false positive calls
  • Samples with > 20% missing genotypes should be excluded
  • Very few variants after filtering (< 100K) - relax filters

6. Phase 6-7: Sample QC and Population Structure

Objective

Apply GWAS-specific quality filters and assess population structure.

Step 6.1: Convert to PLINK Format

plink2 --vcf results/variants/filtered.vcf.gz \
    --make-bed \
    --out results/plink/variants \
    --allow-extra-chr \
    --set-missing-var-ids @:#

Step 6.2: Variant QC

# Apply GWAS QC filters
plink2 --bfile results/plink/variants \
    --maf 0.05 \
    --geno 0.1 \
    --hwe 1e-6 \
    --make-bed \
    --out results/plink/variants_qc \
    --allow-extra-chr

# Save QC summary
plink2 --bfile results/plink/variants_qc \
    --freq \
    --out qc/gwas/allele_freq \
    --allow-extra-chr

plink2 --bfile results/plink/variants_qc \
    --hardy \
    --out qc/gwas/hwe \
    --allow-extra-chr

Step 6.3: Sample QC

# Calculate sample statistics
plink2 --bfile results/plink/variants_qc \
    --missing \
    --out qc/gwas/missing \
    --allow-extra-chr

plink2 --bfile results/plink/variants_qc \
    --het \
    --out qc/gwas/het \
    --allow-extra-chr

# Relatedness check
plink2 --bfile results/plink/variants_qc \
    --make-king-table \
    --king-table-filter 0.0442 \
    --out qc/gwas/relatedness \
    --allow-extra-chr

Step 6.4: PCA for Population Structure

# LD pruning
plink2 --bfile results/plink/variants_qc \
    --indep-pairwise 50 5 0.2 \
    --out qc/gwas/ld_prune \
    --allow-extra-chr

# Calculate PCA
plink2 --bfile results/plink/variants_qc \
    --extract qc/gwas/ld_prune.prune.in \
    --pca 10 \
    --out results/pca/pca \
    --allow-extra-chr
QC CHECKPOINT 4 GWAS-Ready Data Report

Generate Custom MultiQC Module for PLINK

PLINK output isn't natively supported by MultiQC, so we create custom content:

# Create custom MultiQC data files
mkdir -p qc/gwas/multiqc_custom

# Sample missingness for MultiQC
echo -e "Sample\tF_MISS\tStatus" > qc/gwas/multiqc_custom/sample_missingness.txt
awk 'NR>1 {status="PASS"; if($5>0.1) status="FAIL"; print $2"\t"$5"\t"status}' \
    qc/gwas/missing.smiss >> qc/gwas/multiqc_custom/sample_missingness.txt

# Heterozygosity for MultiQC
echo -e "Sample\tO_HET\tE_HET\tF\tStatus" > qc/gwas/multiqc_custom/heterozygosity.txt
# Calculate mean and SD of F coefficient
mean_f=$(awk 'NR>1 {sum+=$6; n++} END {print sum/n}' qc/gwas/het.het)
sd_f=$(awk -v mean=$mean_f 'NR>1 {sum+=($6-mean)^2; n++} END {print sqrt(sum/n)}' qc/gwas/het.het)
awk -v mean=$mean_f -v sd=$sd_f 'NR>1 {
    status="PASS";
    if($6 < mean-3*sd || $6 > mean+3*sd) status="FAIL";
    print $2"\t"$4"\t"$5"\t"$6"\t"status
}' qc/gwas/het.het >> qc/gwas/multiqc_custom/heterozygosity.txt

# Summary statistics
cat > qc/gwas/multiqc_custom/gwas_summary_mqc.yaml << 'EOF'
id: 'gwas_summary'
section_name: 'GWAS QC Summary'
plot_type: 'table'
pconfig:
    id: 'gwas_summary_table'
    title: 'GWAS Quality Control Summary'
data:
EOF

# Add dynamic content
total_variants=$(wc -l < results/plink/variants_qc.bim)
total_samples=$(wc -l < results/plink/variants_qc.fam)
samples_fail_miss=$(awk '$3=="FAIL"' qc/gwas/multiqc_custom/sample_missingness.txt | wc -l)
samples_fail_het=$(awk '$5=="FAIL"' qc/gwas/multiqc_custom/heterozygosity.txt | wc -l)

cat >> qc/gwas/multiqc_custom/gwas_summary_mqc.yaml << EOF
    Total_Variants: $total_variants
    Total_Samples: $total_samples
    Samples_Fail_Missingness: $samples_fail_miss
    Samples_Fail_Heterozygosity: $samples_fail_het
EOF

What to Check

Metric Threshold Action
Sample missingness < 10% Remove samples exceeding threshold
Heterozygosity (F coefficient) Within 3 SD of mean Remove outliers (contamination/inbreeding)
Relatedness (KING) Kinship < 0.0442 (3rd degree) Remove one of related pairs
PCA outliers No extreme outliers Investigate, may indicate different ancestry
HWE (controls only) p > 1e-6 Already filtered in variant QC

8. QC Decision Points Summary

Progressive QC Strategy

Each checkpoint builds on the previous one. STOP the pipeline if critical failures occur.

Checkpoint Critical Failures (STOP) Warnings (Investigate) Continue if
QC1
  • >20% samples with <80% pass rate
  • Systematic adapter contamination
  • High duplication in FastQC
  • Unexpected GC content
 Most samples have Q20+ and >90% pass rate
QC2
  • Mapping rate <80%
  • Mean coverage <5X
  • Duplication >30%
  • Some samples <10X coverage
 Most samples have >95% mapping, target coverage
QC3
  • Ti/Tv <1.5
  • <10K variants after filtering
  • Per-sample missing >10%
  • Ti/Tv <2.0
 Ti/Tv ~2.0+, sufficient variants for GWAS
QC4
  • >30% samples fail QC
  • Strong population structure (confounding)
  • Related individuals
  • PCA outliers
 Sufficient samples after QC, structure controlled by PCs

Final Checklist Before GWAS

Verify the following before running association:

  1. All 4 MultiQC reports generated and reviewed
  2. Samples passing all QC: wc -l results/plink/variants_gwas_ready.fam
  3. Variants passing all QC: wc -l results/plink/variants_gwas_ready.bim
  4. Phenotype file matches sample IDs
  5. PCA covariates file ready
  6. Balanced case/control ratio (or aware of imbalance)
