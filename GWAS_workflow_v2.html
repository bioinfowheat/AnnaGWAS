2. Workflow Overview

Snakemake + micromamba

The entire pipeline is managed by Snakemake with per-rule conda environments resolved by micromamba.

# Dry run
snakemake -n

# Execute (2 test samples, local Mac — 22 of 28 cores)
snakemake --cores 22 --use-conda --conda-frontend micromamba

# Scale to all 94 samples: edit samples.tsv, then re-run
snakemake --cores 22 --use-conda --conda-frontend micromamba

MultiQC Quality Control Strategy

4 QC checkpoints generate MultiQC reports at critical stages:

Checkpoint Stage Tools Aggregated Key Metrics
QC1 Read Quality FastQC, fastp Read quality, adapter content, duplication
QC2 Alignment samtools stats/flagstat/idxstats Mapping rate, coverage, duplicates
QC3 Variants bcftools stats Variant counts, Ti/Tv ratio, quality distribution
QC4 GWAS Ready PLINK 2.0, custom scripts Sample QC, HWE, missingness, PCA

6. Phase 4-5: Variant Calling and Filtering

Objective

Call variants jointly across all samples and apply quality filters to produce a clean set of biallelic SNPs.

Tools & Parameters

StepToolKey Parameters
Pileup bcftools mpileup --min-MQ 20 --min-BQ 20 --annotate FORMAT/AD,FORMAT/DP
Call bcftools call --multiallelic-caller --variants-only
Normalize bcftools norm Left-align, split multiallelic
Filter bcftools view/filter QUAL>20, INFO/DP>5, biallelic SNPs only
Stats bcftools stats Raw + filtered stats for MultiQC

Parallelisation: Variant calling is split per chromosome/scaffold (70 regions), enabling parallel execution across the 22 available cores. Results are concatenated with bcftools concat.

9. QC Decision Points

Checkpoint Critical Failures (STOP) Warnings (Investigate) Continue if
QC1 >20% samples with <80% pass rate; systematic adapter contamination High duplication in FastQC; unexpected GC (not ~36%) Most samples Q20+ and >90% pass
QC2 Mapping rate <80%; mean coverage <5X Duplication >30%; some samples <10X Most samples >95% mapping, target coverage
