2. Workflow Overview

Snakemake + micromamba

The entire pipeline is managed by Snakemake with per-rule conda environments resolved by micromamba.

# Dry run
snakemake -n

# Execute (10 samples, local Mac — 24 of 28 cores)
snakemake --cores 24 --use-conda --conda-frontend mamba

# Scale to all 94 samples: edit samples.tsv, then re-run
snakemake --cores 24 --use-conda --conda-frontend mamba

MultiQC Quality Control Strategy

4 QC checkpoints generate MultiQC reports at critical stages:

Checkpoint Stage Tools Aggregated Key Metrics
QC1 Read Quality FastQC, fastp Read quality, adapter content, duplication

Objective

Align trimmed reads to the edited P. aegeria reference and produce analysis-ready BAM files.

Tools & Parameters

StepToolDetails
Index reference BWA-MEM2 bwa-mem2 index genome.edited.fna
Align BWA-MEM2 PE alignment with read group tags: @RG\tID:{sample}\tSM:{sample}\tPL:ILLUMINA
Sort samtools sort Coordinate sort (piped from BWA-MEM2, -m 10G)
Mark duplicates Picard MarkDuplicates Marks optical & PCR duplicates; rich metrics for MultiQC (library complexity, duplication breakdown)
Index samtools index Create .bai index
Stats samtools stats/flagstat Alignment statistics for MultiQC

Snakemake Rules

rule bwa_mem2_index:    # Index the edited reference genome
rule bwa_mem2_align:    # Align → sort in one piped step (7 threads bwa-mem2 | samtools sort -m 10G)
                        # 3 samples align concurrently (3 × 7 = 21 cores + samtools overhead)
                        # Output: sorted BAM (temp)
rule mark_duplicates:   # Picard MarkDuplicates → dedup BAM + detailed metrics
rule index_bam:         # samtools index → .bai
rule alignment_stats:   # samtools stats + flagstat + idxstats

Performance: BWA-MEM2 output is piped directly to samtools sort — no intermediate unsorted BAM touches disk. With 24 cores and 7 threads per alignment, 3 samples align concurrently.

Disk space: The sorted.bam files are marked as temp() in Snakemake and are automatically deleted once Picard MarkDuplicates completes.
